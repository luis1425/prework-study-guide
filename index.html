HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The !DOCTYPE html declaration tells the browser what document type it should expect when it is opened.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • External style sheet is recommended because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head.
  • Declarations include the CSS property and value of that property
  • Declaration color:blue has the CSS property "color" and the value for that property which is "blue".

Git

  • ls allows us to see list of files and directories on our computer
  • cd command allows us to change directories
  • pwd command will print out directory we are currently in
  • cd.. allows us to go back one directory at a time
  • mkdir bootcamp allowed us to create a new directory called bootcamp
  • touch first-day.txt allowed us to create a new file called first-day.txt
  • git config command to change the name and/or email address associated with your Git commits
  • git revert reverts back to previous code and assists with debugging
  • git commands are not the same as bash commands!!
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add [file]: add a file as it looks now to your next commit (stage)
  • git commit -m "descriptive message"
  • git pull: fetch and merge any commits from the tracking remote branch
  • git push [alias] [branch]: transmit local branch commits to the remote repository branch
  • git pull origin main: to pull in lastest changes to main branch (after switching to main branch)

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Example of variable: var shapes = ["circle", "square", "triangle"];
  • console.log(shapes[1]); to see the string square in console
  • for(var x = 0; x < shapes.length; x++) { console.log(shapes[x]); }
  • All strings are surrounded by quotation marks.
  • Each line of JavaScript code ends with a semicolon.
  • if (first condition) { block of code to be executed if first condition is truthy; }
  • else if (second condition) { block of code to be executed if second condition is truthy; }
  • var studentInfo = ["Lu", 54, true];
  • "Lu" is a string, 54 is a number, and true is a Boolean value so they do not require quotations
  • Arrays are useful tools for storing multiple items with different data types in a single variable.
